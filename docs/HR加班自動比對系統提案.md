## 一、我們理解的現況

### 你每月要做的事

```
每月流程：

  ① 從人資系統下載「出勤明細」
     → 系統自動標記所有打卡 < 8:00 的人為「早到加班 0.5H」
     → 但很多人只是習慣早到，並沒有真的在加班

  ② 從生管系統下載「工時統計單」
     → QR CODE 掃碼時間才能反映員工是否真的開始工作
     → QR < 8:00 = 確實在加班
     → QR ≥ 8:01 = 沒在加班，只是人到了

  ③ 收集紙本「早上加班申請單」
     → 有主管核章才算正式報加班

  ④ 手動逐筆比對上面三份資料
     → 每月約 250+ 筆「早到」紀錄需要逐一檢查
     → 判斷哪些是真正加班、哪些只是習慣早到
     → 同時處理晚班加班的統計

  ⑤ 把確認後的加班時數登記回系統

  ⑥ 產出最終報表（加班46hr分段）
```

### 目前的痛點

| 問題 | 說明 |
|------|------|
| 工作量大 | 每月 250+ 筆早到紀錄要逐一比對 |
| 重複性高 | 每月做完全相同的比對動作 |
| 容易疲勞出錯 | 大量數字肉眼比對，漏看或判斷錯很正常 |
| 核心原因 | 出勤系統太「傻」— 只要打卡 < 8:00 就自動算加班，不管有沒有真的在工作 |

## 四、判定規則說明

### 早班加班（8:00~8:30 時段）

```
判定條件：
  ① 出勤打卡 < 08:00（人有到）
  ② QR CODE < 08:00（確實開始工作）
  ③ 以上兩者都成立 → 算 0.5H 早班加班

排除條件：
  → 出勤打卡 < 08:00，但 QR CODE ≥ 08:00
  → 代表人雖然到了，但沒有真的開始工作
  → 系統自動排除，不算加班
```

#### 用實際案例說明

**案例 A — 小安 (賴葦蓁 50028) 4/13：算加班 ✓**

```
  出勤打卡：07:44 ← 人到了
  QR CODE：07:56  ← 開始工作了（< 8:00）
  結果：算 0.5H 早班加班
  日工時小計：8.0 + 0.5 = 8.5
```

**案例 B — 陶阿姨 (陶玉香 30010) 4/13：不算加班 ✗**

```
  出勤打卡：07:?? ← 人到了
  QR CODE：08:08  ← 8點多才開始工作（≥ 8:00）
  結果：不算加班，系統自動排除
  （出勤系統會跳 0.5H，但這是錯誤的）
```

**案例 C — 邊界案例 4/16：待確認 ⚠️**

```
  出勤打卡：07:50 ← 人到了
  QR CODE：08:02  ← 差 2 分鐘超過 8:00
  結果：系統標記「待確認」，由你判斷
  （可能是不小心晚刷）
```

### 晚班加班（18:00 以後）

```
判定條件：
  → 出勤下班打卡 > 18:00
  → 加班時數 = 下班時間 - 18:00（依實際時間計算）
  → 不需要加班申請單

分級計算：
  → 18:00 ~ 20:00 之間 = 平日加班 1-2H
  → 20:00 ~ 22:00 之間 = 平日加班 3-4H
  → 超過 22:00        = 平日加班 >4H

假日加班：
  → 休息日/例假日有出勤 = 依實際工時計算
```
